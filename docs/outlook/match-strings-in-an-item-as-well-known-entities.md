<span data-ttu-id="83037-183">Opcionalmente, você pode aplicar uma expressão regular em `ItemHasKnownEntity` uma regra, para que as instâncias de filtro sejam mais futuras de uma entidade e fazer com que o Outlook ative um suplemento somente em um subconjunto das instâncias da entidade.</span><span class="sxs-lookup"><span data-stu-id="83037-183">You can optionally apply a regular expression in an `ItemHasKnownEntity` rule, so as to further filter instances of an entity and have Outlook activate an add-in only on a subset of the instances of the entity.</span></span> <span data-ttu-id="83037-184">Por exemplo, você pode especificar um filtro para a entidade de rua do endereço em uma mensagem que contenha um CEP do Rio de Janeiro que comece com "021".</span><span class="sxs-lookup"><span data-stu-id="83037-184">For example, you can specify a filter for the street address entity in a message that contains a Washington state zip code beginning with "98".</span></span> <span data-ttu-id="83037-185">Para aplicar um filtro nas instâncias de entidade, use os `RegExFilter` atributos `FilterName` e no `Rule` elemento do tipo [ItemHasKnownEntity](../reference/manifest/rule.md#itemhasknownentity-rule) .</span><span class="sxs-lookup"><span data-stu-id="83037-185">To apply a filter on the entity instances, use the `RegExFilter` and `FilterName` attributes in the `Rule` element of the [ItemHasKnownEntity](../reference/manifest/rule.md#itemhasknownentity-rule) type.</span></span>

```XML
   <Rule xsi:type="ItemIs" ItemType="Message" />
   <Rule xsi:type="ItemHasKnownEntity" EntityType="Address" />
```

<span data-ttu-id="83037-189">O exemplo a seguir `getEntitiesByType` usa o item atual para definir uma variável `addresses` com os resultados da coleção de regras anterior.</span><span class="sxs-lookup"><span data-stu-id="83037-189">The following example uses `getEntitiesByType` of the current item to set a variable `addresses` to the results of the preceding rule collection.</span></span>


<br/>

<span data-ttu-id="83037-190">O exemplo `ItemHasKnownEntity` de regra a seguir ativa o suplemento sempre que houver uma URL no assunto ou no corpo do item atual, e a URL contiver a cadeia de caracteres "YouTube", independentemente do caso da cadeia de caracteres.</span><span class="sxs-lookup"><span data-stu-id="83037-190">The following `ItemHasKnownEntity` rule example activates the add-in whenever there is a URL in the subject or body of the current item, and the URL contains the string "youtube", regardless of the case of the string.</span></span>

```XML
<Rule xsi:type="ItemHasKnownEntity" 
    EntityType="Url" 
    RegExFilter="youtube"
    FilterName="youtube"
    IgnoreCase="true"/>
```

<br/>

<span data-ttu-id="83037-191">O exemplo a seguir `getFilteredEntitiesByName(name)` usa o item atual para definir uma variável `videos` para obter uma matriz de resultados que correspondam à expressão regular na `ItemHasKnownEntity` regra anterior.</span><span class="sxs-lookup"><span data-stu-id="83037-191">The following example uses `getFilteredEntitiesByName(name)` of the current item to set a variable `videos` to get an array of results that match the regular expression in the preceding `ItemHasKnownEntity` rule.</span></span>

